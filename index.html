Smart Campus Navigator

An IoT Wayfinding Device for UC Davis

Project Overview

The Smart Campus Navigator is an Internet-of-Things navigation device designed to guide users to locations across the UC Davis campus without using a GPS module directly on the device. Instead, GPS coordinates are obtained from a mobile device and transmitted through a cloud backend to an embedded system built around the Texas Instruments CC3200 microcontroller.

The system combines cloud computing, wireless networking, embedded programming, and sensor fusion to determine the user's orientation and display a compass-style navigation interface on an OLED display.

Key Features

  • Cloud-connected IoT architecture
  • Real-time directional navigation
  • Compass interface on OLED display
  • IR remote menu navigation
  • Magnetic heading using BNO055 IMU

Technologies Used

  • CC3200 Wi-Fi Microcontroller
  • AWS IoT Device Shadow
  • AWS Lambda backend
  • SPI and I2C communication
  • Interrupt-driven IR decoding
  • HTML + JavaScript frontend

System Architecture

The system architecture consists of three layers: the mobile frontend, cloud backend, and embedded device.

Mobile Frontend

A web application running on the user's smartphone retrieves the device's GPS coordinates and sends them to an AWS endpoint every five seconds. The web application is built using HTML and JavaScript to send the latitude and longitude in JSON format to be directly updated in the shadow.

Cloud Backend

AWS Lambda processes the coordinates and updates the AWS IoT Device Shadow, allowing the embedded device to retrieve the data. The Lambda function is designed with Python to behave like an interrupt; it updates the shadow every time a change is detected in the endpoint.

Embedded System

The CC3200 microcontroller retrieves coordinates from AWS via Wi-Fi, reads heading data from the BNO055 magnetometer sensor over I2C, and calculates the distance and bearing to the target. The direction is displayed on the OLED compass interface via SPI.


Hardware Prototype

The prototype integrates multiple embedded peripherals connected to the CC3200 LaunchPad development board.

CC3200 LaunchPad

Main microcontroller handling Wi-Fi communication and system control. It allowed us to implement the state machine described below. In addition, it includes a UART system that allowed us to send/receive JSON packages to/from AWS to monitor our shadow, specifically the user's coordinates.

BNO055 IMU

Provides orientation data used to determine the user's heading. Using I2C, we can wait for the calibration mask to fill up before entering program execution, then read using the magnetometer's mask.

OLED Display

SPI-driven display used to render compass graphics and distance data. We used a given library of functions that allowed us to draw circles, lines, and text.

IR Receiver

Allows the user to navigate the menu using a handheld IR remote. The IR Receiver is wired into a GPIO pin on the CC3200. The software includes a GPIO interrupt that measures the time between falling edges on the IR Receiver. Since the remote signal encoding is always a single 0 followed by some number of 1s, we can divide the time past by the bit-sending frequency of the remote to decode the signal.


Bill of Materials

The following components were used to build the Smart Campus Navigator prototype. Many parts were provided through the EEC172 lab, while others were inexpensive sensors and wiring used to assemble the breadboard prototype.

CC3200 LaunchPad

The TI CC3200 LaunchPad acts as the main microcontroller and Wi-Fi networking platform for the system.

Cost: Provided by lab

BNO055 IMU Sensor

A 9-DOF inertial measurement unit used to determine the user’s orientation and heading for the compass navigation system.

Cost: ~$35

SSD1351 OLED Display

SPI-based OLED display used to render the compass interface and navigation information.

Cost: Provided by lab

IR Receiver Module

Infrared receiver used to decode signals from a handheld remote for navigating the device menu.

Cost: Provided by lab

IR Remote Control

Used by the user to select destinations and interact with the menu system displayed on the OLED.

Cost: Provided by lab

Jumper Wires / Breadboard

Used to connect the CC3200 board to the sensor modules and display during prototyping.

Cost: ~$5


Total Estimated Cost: ~$40

Results

Accurate Navigation

The system successfully computes the relative bearing between the user and destination and renders an intuitive compass interface.

Cloud Integration

Real-time coordinate updates are transmitted through AWS IoT, demonstrating a full edge-to-cloud IoT architecture.

Responsive Display

Selective OLED redraw prevents flickering and enables smooth compass animation during navigation.
